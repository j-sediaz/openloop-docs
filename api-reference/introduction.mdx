## Welcome to Partners API

The Partners API is a comprehensive patient management system designed for healthcare applications. It provides secure endpoints for managing patient data, including personal information, contact details, and medical records.

## Key Features

- **Patient Management**: Create, read, update, and delete patient records
- **Search & Filtering**: Advanced search capabilities across patient data
- **Data Validation**: Comprehensive validation using JSON Schema
- **Authentication**: Secure API access with multiple authentication methods
- **System Monitoring**: Health check and API information endpoints

## Available Endpoints

### Patient Management
- **List Patients**: `GET /patients` - Retrieve all patients with search and pagination
- **Create Patient**: `POST /patients` - Add new patient records
- **Get Patient**: `GET /patients/{id}` - Retrieve specific patient by ID
- **Update Patient**: `PUT /patients/{id}` - Update patient data

### System Endpoints
- **Health Check**: `GET /health` - Monitor API status and connectivity
- **API Information**: `GET /` - Get system information and available endpoints

## Authentication

The Partners API supports two authentication methods:

### API Key Authentication
```bash
curl -H "x-api-key: your-api-key" \
  https://partners-api-sigma.vercel.app/patients
```

### Bearer Token Authentication
```bash
curl -H "Authorization: Bearer your-token" \
  https://partners-api-sigma.vercel.app/patients
```

## Base URL

- **Production**: `https://partners-api-sigma.vercel.app`

## Rate Limits

- **Standard**: 1000 requests per hour
- **Burst**: 100 requests per minute

## Getting Started

1. **Obtain API Credentials**: Contact your administrator for API keys
2. **Test Connection**: Use the health check endpoint to verify connectivity
3. **Explore Endpoints**: Start with listing patients to understand the data structure
4. **Create Patients**: Use the create endpoint to add new patient records

<Note>
  This API is designed for healthcare applications and includes sensitive patient data. Ensure you follow all applicable privacy regulations (HIPAA, GDPR, etc.) when implementing.
</Note>
