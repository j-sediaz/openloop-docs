## Overview

OpenLoop organizes healthcare services into **Clinical Verticals** (bundles) containing related programs. Each program represents a specific condition or treatment area with its own questionnaire and associated products.

<Info>
Programs use **abbreviation IDs** (e.g., ADHD, MHL, ED) as unique identifiers across the API.
</Info>

## Working with Programs

### Listing Programs

<CodeGroup>
```bash Get All Programs
curl -H "x-api-key: YOUR_API_KEY" \
     https://api-staging.openloophealth.com/programs
```

```bash Filter by Bundle
curl -H "x-api-key: YOUR_API_KEY" \
     "https://api-staging.openloophealth.com/programs?bundle_code=BH"
```

```bash Filter by Gender
curl -H "x-api-key: YOUR_API_KEY" \
     "https://api-staging.openloophealth.com/programs?bundle_code=SH&gender=M"
```
</CodeGroup>

### Getting Program Details

```bash
curl -H "x-api-key: YOUR_API_KEY" \
     https://api-staging.openloophealth.com/programs/ADHD
```

### Program Workflow

<Steps>
<Step title="Explore Programs">
  Client discovers available programs by bundle or gender filters.
  
  ```bash
  GET /programs?bundle_code=DERM
  ```
</Step>

<Step title="Get Questionnaire">
  Retrieve the program-specific questionnaire for patient assessment.
  
  ```bash
  GET /programs/MHL/questionnaires
  ```
</Step>

<Step title="View Products">
  See available products (pricing tiers, frequencies) for the program.
  
  ```bash
  GET /programs/MHL/products
  ```
</Step>

<Step title="Create Order">
  Patient completes questionnaire and selects a product to create an order.
  
  ```bash
  POST /orders
  ```
</Step>
</Steps>

## Program Response Schema

```json
{
  "id": "ADHD",
  "name": "ADHD",
  "bundle_code": "BH",
  "bundle_name": "Behavioral Health",
  "offering_name": "ADHD",
  "age_range": null,
  "gender": null
}
```

<ResponseField name="id" type="string" required>
  Program abbreviation ID used throughout the API
</ResponseField>

<ResponseField name="bundle_code" type="string" required>
  Clinical vertical code (BH, DERM, SLEEP, etc.)
</ResponseField>

<ResponseField name="gender" type="string">
  Gender restriction: `M` (Male), `F` (Female), `ALL`, or `null` (no restriction)
</ResponseField>

## Best Practices

<Tip>
Cache the programs list client-side as it changes infrequently. Refresh daily or on app updates.
</Tip>

<Warning>
Always check the `gender` field before displaying programs to users to avoid showing gender-specific programs to the wrong audience.
</Warning>

<Note>
Programs with `active: false` should not be displayed to end users but may appear in historical data.
</Note>
